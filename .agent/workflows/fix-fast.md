Direct fix with minimal overhead for simple bugs.

## Workflow

1. **Analyze** the issue description
2. **Locate** the affected file(s)
3. **Fix** directly
4. **Verify** the fix compiles/runs
5. **Report** briefly

## Notes
- No research or planning phase
- Single-file or few-file fixes
- Best for straightforward bugs
