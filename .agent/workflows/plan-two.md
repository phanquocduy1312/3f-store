Create two alternative implementation approaches for comparison.

## Workflow

1. **Research** the problem space
2. **Scout** codebase
3. **Design** two distinct approaches:
   - **Approach A**: [Describe with pros/cons]
   - **Approach B**: [Describe with pros/cons]
4. **Compare** both approaches (effort, risk, performance, maintainability)
5. **Create** plan in `./plans/` with both approaches documented
6. **Ask** user to choose preferred approach

## Report Output

Save to `plans/{YYMMDD}-{HHMM}-{slug}/`:
- `research/researcher-XX-{topic}.md` — ≤150 lines each
- `plan.md` — overview with both approaches and comparison table
- `phase-XX-{name}.md` — detailed phase files for chosen approach

## Notes
- Always present meaningful alternatives (not trivially different)
- Include clear trade-off comparison
- **Do not** start implementing
