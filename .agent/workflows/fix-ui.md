Debug and fix visual/interaction issues in the UI.

## Workflow

1. **Analyze** the reported UI issue
2. **Take screenshot** if possible (using `chrome-devtools` skill)
3. **Identify** the CSS/component causing the issue
4. **Fix** the UI code
5. **Verify** visually (screenshot comparison)
6. **Test** responsive behavior
7. **Report** changes

## Notes
- Use `chrome-devtools` skill for screenshots and DOM inspection
- Check responsive layouts at 320px, 768px, 1024px, 1440px
- Activate `ui-styling` or `frontend-design` skills as needed
