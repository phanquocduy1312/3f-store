Create a new skill in the skills directory.

## Workflow

1. **Clarify** requirements (ask questions if needed)
2. **Research** the topic (documentation, GitHub repos, APIs)
3. **Plan** skill structure (SKILL.md, references, scripts, assets)
4. **Create** skill directory and files:
   - SKILL.md (under 100 lines, YAML frontmatter)
   - `references/` for detailed docs
   - `scripts/` for executable code
5. **Test** scripts
6. **Report** what was created

## Notes
- Skills teach how to perform tasks, not what tools do
- Progressive disclosure: SKILL.md → references → scripts
- Keep SKILL.md and references under 100 lines each
