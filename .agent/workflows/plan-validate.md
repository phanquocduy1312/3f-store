Review an existing plan with probing questions to ensure completeness.

## Workflow

1. **Read** the plan file
2. **Generate** 3-8 critical validation questions covering:
   - Missing requirements
   - Unstated assumptions
   - Security concerns
   - Performance implications
   - Edge cases
   - Integration risks
3. **Ask** questions to the user one at a time
4. **Update** plan based on answers
5. **Report** validation status

## Notes
- Questions should challenge assumptions
- Focus on what could go wrong
- Update plan with new insights
